从0开始搭建一个环境虚拟环境

新建一个虚拟环境

conda create --name env_name packages_name

–name选项可简化为-n，指出要创建的环境名称，并可以在创建环境时安装包。新建环境后根据需要pip install或者conda install安装就好。

使用打包的方式从一台服务器的虚拟环境上迁移到另一台服务器上

这个现实就是我们的服务器一共有几台，得紧盯着一没人就赶紧把站卡占上，相同配置的服务器有多个，如何在在多个相同环境的服务器下迁移环境呢。

安装conda-pack

conda install -c conda-forge conda-pack
# 这里设置为default的话就不用 -c了，如果使用清华源推荐的方式的话，不加下不下来
# pip install git+https://github.com/conda/conda-pack.git

PS: 下面就是清华源推荐的配置方式, 我们直接将custom_default填到default_channels就不用-c指定channel了，但是conda-forge更新时有问题。

channels:
   - defaults
 show_channel_urls: true
 default_channels:
   - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main
   - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/r
   - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/msys2
 custom_channels:
   conda-forge: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud
   msys2: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud
   bioconda: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud
   menpo: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud
   pytorch: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud
   simpleitk: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud

打包已经安装好的环境

conda pack -n my_env_name -o my_env_name.tar.gz

拷贝打包好的环境my_env_name.tar.gz到目标机器，并解压到任何目录，一般推荐放到envs目录下 (anaconda_root/envs)。(注意：anaconda_root改为自己的conda安装路径。)

# 解压打包好的环境
# 默认是全都解压到当前目录，场面很壮观
# -C 一定要指定
mkdir -p anaconda_root/envs/my_env
tar -xzf my_env.tar.gz -C anaconda_root/envs/my_env

# 激活环境
source my_env/bin/activate
 
# Unpack
conda-unpack
 
# 至此环境就完全拷贝过来了
 
# 去激活
source deactivate

修改开源代码的环境为自己的环境

一般情况下，深度学习开源代码的环境和我们的环境经常是不同的。因为pytorch版本是不同的，pytorch一般一个版本只支持几个版本的cuda，nvidia-driver和cuda有对应关系,如果必须使用这个pytorch的版本，cuda不支持，这是无解的一个问题，只能去重装驱动。PS：cuda的版本不要用nvidia-smi命令来查cuda版本，是错的，应该用cat来查看。我们只能尽量的去手工解决版本不同的问题。下面举个例子来解释一下我是怎么处理环境冲突的。
开源代码向我们提供的有可能是：
__参考博客

  • pip导出的requestment.txt文件
    pip freeze > requirements.txt
  • conda导出的environment.yaml文件
    conda env export > environment.yaml

我们想安装别人虚拟环境的时候就这样：

  • 通过requestment.txt
    pip -r requirements.txt
  • 或者是environment.yaml
    conda env create -f environment.yaml
    大部分情况下，要以我们的硬件cuda版本为主，pytorch的版本要做出妥协，这时候pytorch的版本需要做一定的改变，同时会产生冲突，需要我们手动去解决这些冲突。举个栗子：搭建cosypose的环境。
    官方给出的cosypose环境是：
    - pytorch=1.3.1=py3.7_cuda10.1.243_cudnn7.6.3_0
- torchvision=0.4.2=py37_cu101
    服务器环境是cuda11.0，注意这里cuda的版本是10.1，但是pytorch1.3.1是没有二进制包支持cuda11.1的，我们将这个包替换成了支持cuda11.0版本的pytorch=1.7.1=py3.8_cuda11.0.221_cudnn8.0.5_0，我是怎么知道这个名字并使用conda install pytorch=1.7.1=py3.8_cuda11.0.221_cudnn8.0.5_0来进行安装的呢。
    conda search package_name
    会出现一堆带有后缀名称的包，然后挑选cuda11.0的和合适python版本的改到environment.yaml中，再执行conda env create -f environment.yaml进行安装，这是候因为版本冲突，一般会报错的，这时候我们使用：
    conda info package_name # 查看指定包，其需要的依赖
    或者
    pipdeptree -p package_name # 查看指定包，其需要的依赖
pipdeptree -p package_name -r # 查看哪些其它包，依赖于此指定包
    查看依赖关系，手动解决冲突（就是更换版本），比如这里torchvision就出现了冲突，我选择了更高版本的解决了冲突。直到conda create -f -n environment.yaml和pip install -r requirements.txt完成，我们的环境就搭建好了，可以进行代码的debug和改数据集进行消融实验，加、改网络提点了。
